# `src/infra`

- Purpose: Infrastructure-oriented source assets used by runtime and operations.
- Scope: Folder path is `src/infra`.
- Should contain: Should contain implementation files, schemas, tests, or docs directly relevant to `infra`.
- Should not contain: Should not contain unrelated domain logic or artifacts owned by sibling folders under `src/`.
- Keep files here focused, small, and aligned with this folder contract.
