# People, HR, and Innovation (human lifecycle)

Compiles to persona YAML stages -> steps with staffing reliability and controlled process improvement.

## Always-on rules
- Critical roles must have backup coverage.
- No process change without pilot scope and success metric.
- Training completion is mandatory for role handoffs.
- Failed experiments must be documented and retired.

## Phase process (intern-safe, anti-bug)
- Phase 20 - Capacity planning. Actions: review demand forecast and current coverage, update staffing model template, verify owner/backup coverage. Gate: staffing model approved. Artifacts: workforce plan.
- Phase 20 - Capability development. Actions: run cross-training schedule, execute SOP onboarding checklist, verify readiness assessment pass. Gate: role readiness pass. Artifacts: training completion log.
- Phase 20 to 21 - Innovation control. Actions: define pilot scope/metric, run limited experiment, compare outcome vs success metric, verify ship/rollback decision. Gate: experiment success or rollback decision. Artifacts: pilot report.
- Cross-phase support - Resilience checks. Actions: audit workflows for single-point failures, assign backup owners, verify coverage matrix completeness. Gate: backup owner assigned per critical workflow. Artifacts: resilience matrix.

## Daily execution checklist
1. Browse staffing, training, and experiment trackers for gaps.
2. Update HR and SOP systems immediately after each action.
3. Run approved reports/automations only; avoid ad-hoc code edits.
4. Perform gate review with second approver for role changes.
5. Escalate critical coverage gaps same day.
