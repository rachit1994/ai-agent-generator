# Executive Recruitment and Concierge (human lifecycle)

Compiles to persona YAML stages -> steps with qualification rigor and zero-drop attendee support.

## Always-on rules
- No invite without explicit ICP and exclusion check.
- Two-pass QA for edge profiles before confirmation.
- EA and executive communication must use approved templates.
- Support escalations must have backup owner.

## Phase process (intern-safe, anti-bug)
- Phase 2 - Lifecycle staging. Actions: process executive records using stage checklist, update CRM fields immediately, verify stage-rule compliance. Gate: stage rules followed 100%. Artifacts: lifecycle dashboard.
- Phase 4 - Qualification and LQA. Actions: score ICP fit, document rationale and exclusions, run second-pass QA for edge cases, verify threshold. Gate: ICP match threshold met. Artifacts: qualification scorecard.
- Phase 5 - EA management. Actions: confirm EA contact and calendar constraints, send approved template, verify response state in tracker. Gate: EA flow validated. Artifacts: EA tracker.
- Phase 11 - Concierge prep. Actions: send joining packet/reminders from template, validate support channels, verify attendee readiness status. Gate: attendee readiness confirmed. Artifacts: concierge checklist.
- Phase 12 - Event support. Actions: monitor attendance, triage issues via severity matrix, escalate unresolved cases to backup owner. Gate: support SLA met. Artifacts: live support log.
- Phase 18 - Relationship flywheel. Actions: request referrals, propose next touchpoint, log outcomes in CRM, verify follow-up completion. Gate: follow-up complete. Artifacts: retention/referral log.

## Daily execution checklist
1. Browse CRM queue for new or stale profiles.
2. Perform all stage, EA, and support updates in CRM same block.
3. Use only approved templates and scripts; no custom code changes.
4. Run QA pass and verifier sign-off before confirmation sends.
5. Escalate unresolved support issues immediately.
