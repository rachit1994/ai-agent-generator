# Phase 7: Compliance & Security

[← Phase 6](phase-06.md) | [Overview](00-overview.md) | [Phase 8 →](phase-08.md)

---

## Purpose

Embed privacy and security from design to ensure trust and regulatory adherence.

## Teams

Legal, Security.

## Tools

- OneTrust (consent workflows).
- Osano (DSAR handling).
- DocuSign (agreements).
- AWS KMS (data encryption).
- Okta (identity federation).

## Process

```
Risk assessment (per event/topic)
↓
Consent capture (opt-in forms)
↓
Access provisioning (role-based)
↓
Audit logging (immutable trails)
↓
Breach simulation (quarterly drills)
```

## Deliverables

- Consent records database (revocable).
- Security policies playbook.
- Compliance logs (SOC 2 aligned).

## Metrics

| Metric               | Target |
|----------------------|--------|
| Compliance incidents | 0      |

## Success Criteria

100% audit pass rate; executives rate trust >9/10.
