# Personas (specs for YAML generation)

These docs define **persona behavior** as **phases → steps** (human lifecycle), written to be directly mappable to persona YAML **stages → steps**.

- The persona library is the product; these are **requirements** it must be able to express and enforce.
- Prefer **gates** (pass/fail) and **artifacts** over prose.
