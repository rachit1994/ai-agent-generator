# Event network roles (derived from `docs/phases`)

This persona set is generated from the exact team names and ownership model in `docs/phases` (especially `14-day-lifecycle-daily-tasks.md` and `00-overview.md`).

Each role below maps to:
- **Primary phases** it is accountable for
- **Core process** it follows
- **Gate** that must pass before handoff
- **Artifacts** it must produce

## Always-on rules (all roles)

- Clarification first: no execution on ambiguous scope.
- Evidence-first decisions: every gate decision links to artifact proof.
- Two-person control for risky changes (data, legal, security, production).
- Handoffs require owner, due date, and success definition.
- Post-event learning is mandatory and captured in reusable logs.
