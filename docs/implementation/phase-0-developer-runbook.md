# Phase 0 — developer and CI prerequisites (machines and repo ready)

This runbook satisfies **Phase 0** items **1–5** in `docs/implementation/2026-04-13-persona-agent-platform-phased-checklist.md` with **machine-checkable** pointers (item **3** also requires environment-specific filled pins for a full “done” story). **Items 6–7 (legal-name DRIs)** remain **human-assigned**; see `docs/implementation/phase-0-owner-assignments.md`.

## 1) Python 3.13+ and pinned lockfile

- **Interpreter pin:** `.python-version` → `3.13` (matches `requires-python = ">=3.13"` in `pyproject.toml`).
- **Lockfile:** `uv.lock` (generated by `uv lock`; install with `uv sync --frozen`).
- **Evidence (local):** `uv run python -V` prints **3.13.x**; `uv sync --frozen` exits **0**.

## 2) Postgres + `pgvector` and migration path

- **Local database:** `docker compose up -d` using `docker-compose.yml` (`pgvector/pgvector:pg16`).
- **CI database:** `.github/workflows/ci.yml` runs the same `pgvector/pgvector:pg16` image as a service container; the workflow enables `vector` with `CREATE EXTENSION IF NOT EXISTS` then asserts the extension row exists (proves **both** developers and CI reach Postgres + pgvector per checklist item 2).
- **Extension enablement:** first-init SQL in `migrations/docker-init/01-enable-pgvector.sql` (`CREATE EXTENSION IF NOT EXISTS vector;`).
- **Rollout story:** `migrations/README.md` (extension first; app DDL later via a versioned migrator once tables exist).

## 4) CI: `promptfoo`, `pytest` + DeepEval, OpenTelemetry

- **Workflow:** `.github/workflows/ci.yml` runs `uv sync --frozen`, `uv run pytest`, and `npx promptfoo@0.118.0 eval -c promptfooconfig.yaml`.
- **DeepEval:** `tests/test_deepeval_smoke.py` exercises `deepeval.assert_test` with a deterministic metric (no external LLM keys).
- **OpenTelemetry:** `tests/test_otel_export_smoke.py` and `tests/test_otel_otlp_exporter_smoke.py` prove span capture and OTLP HTTP exporter wiring for later **agentevals** integration.
