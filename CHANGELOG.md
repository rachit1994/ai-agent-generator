# Changelog

## 0.12.0 — 2026-04-19

### Breaking (repo-only imports)

Public **Python import strings** from the old flat `src/` layout are **not** preserved. Update imports to the new tree:

| Old | New |
|-----|-----|
| `sde_foundations.*` | `storage.storage`, `common.utils`, `sde_types.types`, `artifacts.artifacts`, `sde_eval.eval`, `model_adapter.model_adapter`, `safeguards.safeguards` (under repo-root `libs/`) |
| `sde_gates` | `guardrails_and_safety` (facade) or `guardrails_and_safety.risk_budgets.*` / `review_gating.*` / `autonomy_boundaries_tokens_expiry.*`; shared pieces in `gates_constants`, `gates_manifest`, `time_and_budget` |
| `sde_pipeline` | `workflow_pipelines.production_pipeline_task_to_promote` |
| `sde_modes` | `workflow_pipelines.execution_modes` |
| `orchestrator` | unchanged **import name**; on-disk path is `src/production_architecture_what_runs_on_the_laptop/orchestrator/` |

See [`docs/architecture/repository-layout-from-completion-inventory.md`](docs/architecture/repository-layout-from-completion-inventory.md) and [`docs/architecture/adr-gates-hs-row-mapping.md`](docs/architecture/adr-gates-hs-row-mapping.md).

### Packaging

- Hatch `packages` and `dev-mode-dirs` now cover `src/` section roots, `libs/`, and the extra path entry so `import orchestrator` resolves.
- `src_legacy/` removed after cutover.
